<ParamField query="namespace" type="string">
  Filter by Kubernetes namespace
</ParamField>

<ParamField query="state" type="string">
  Filter by state: `Pending`, `Executing`, `Verifying`, `Completed`, `Failed`, `RolledBack`
</ParamField>

<ParamField query="issue" type="string">
  Filter by incident name (e.g., `INC-20260319-001`)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="pageSize" type="integer" default="20">
  Items per page (max 100)
</ParamField>

## Remediation Modes

Each plan operates in one of two modes:

- **Runbook mode** (`agenticMode: false`): Executes a pre-defined sequence of actions from a matched runbook
- **Agentic mode** (`agenticMode: true`): AI-driven step-by-step remediation where the LLM proposes actions, observes results, and decides next steps
