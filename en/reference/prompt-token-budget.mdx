Every ChatCLI session starts by sending a **system prompt** to the model. That prompt is not monolithic: it's assembled dynamically from several blocks (mode instructions, workspace context, plugin schemas, scratch dir hint, agent catalog, active persona) — and the final size varies a lot between `chat`, `/agent`, and `/coder` modes.

This page empirically measures every block in the current code, sums the total cost per mode, and shows how **prompt caching** completely flips the economics for sessions of more than one turn.

<Info>
The numbers below reflect the current `main` branch of `chatcli`. Token estimates use the standard rule of **4 characters per token** (typical accuracy of ±10% versus real tiktoken on mixed text/code). To reproduce the values, see [How to reproduce](#how-to-reproduce).
</Info>

---

## System prompt anatomy

ChatCLI assembles the system prompt in layers, defined in `cli/prompts.go`, `cli/cli_llm.go`, and `cli/agent_mode.go`:

```text
┌─────────────────────────────────────────────────────────┐
│ Workspace context (AGENT.md, SOUL.md, USER.md, ...)     │
├─────────────────────────────────────────────────────────┤
│ Main mode block:                                        │
│   - chat   -> ChatModeSystemHint                        │
│   - /coder -> CoderSystemPrompt (or CoderFormat+persona)│
│   - /agent -> agent.system_prompt.default.base          │
├─────────────────────────────────────────────────────────┤
│ ai.response_language (locale instruction)               │
├─────────────────────────────────────────────────────────┤
│ getToolContextString (plugin schemas)                   │
├─────────────────────────────────────────────────────────┤
│ buildSessionWorkspaceHint (CHATCLI_AGENT_TMPDIR + tips) │
├─────────────────────────────────────────────────────────┤
│ OrchestratorSystemPrompt (multi-agent catalog)          │
└─────────────────────────────────────────────────────────┘
```

`chat` mode receives only the first two layers; `/agent` and `/coder` receive all of them.

## Per-block table

Each block measured in isolation, with its contribution in characters and tokens:

| Block | When it ships | Chars | ~Tokens |
|:------|:--------------|------:|--------:|
| `ChatModeSystemHint` | chat only | 855 | **214** |
| `i18n("ai.response_language")` (PT) | always | 306 | **77** |
| `CoderSystemPrompt` | `/coder` without persona | 6,548 | **1,637** |
| `CoderFormatInstructions` | `/coder` with active persona | 2,207 | **552** |
| `AgentFormatInstructions` | `/agent` with active persona | 1,259 | **315** |
| `agent.system_prompt.default.base` (PT) | `/agent` without persona | 2,905 | **727** |
| `getToolContextString` (3 built-in plugins) | `/agent` + `/coder` | 8,628 | **2,157** |
| `buildSessionWorkspaceHint` | `/agent` + `/coder` | 1,593 | **399** |
| `OrchestratorSystemPrompt` (12 default agents) | `/agent` + `/coder` with `CHATCLI_AGENT_PARALLEL_MODE=true` | 16,290 | **4,073** |
| `BuildSystemPromptPrefix` (workspace context) | always | variable | depends on project |

`getToolContextString` grows as you add external plugins. `BuildSystemPromptPrefix` depends on which context files exist in the project (`AGENT.md`, `SOUL.md`, `USER.md`, `MEMORY.md`, `IDENTITY.md`, `RULES.md`).

## Total cost per mode

Comparison of the **initial spend** in each mode, assuming default config (no persona, multi-agent on, no workspace prefix). Numbers are tokens sent in the first message of the session.

### `chat` mode

```text
ChatModeSystemHint           214
ai.response_language          77
                           -----
TOTAL                       ~291 tokens
```

### `/agent` default (no persona, multi-agent ON)

```text
agent.system_prompt.default.base    727
ai.response_language                 77
getToolContextString              2,157
buildSessionWorkspaceHint           399
OrchestratorSystemPrompt          4,073
                                  -----
TOTAL                            ~7,433 tokens
```

### `/coder` (no persona, multi-agent ON)

```text
CoderSystemPrompt                 1,637
ai.response_language                 77
getToolContextString              2,157
buildSessionWorkspaceHint           399
OrchestratorSystemPrompt          4,073
                                  -----
TOTAL                            ~8,343 tokens
```

### With active persona (`/agent` or `/coder`)

The persona replaces the main fixed prompt:

```text
personaPrompt                     (~800 to 3,000)
+ AgentFormatInstructions           315       (in /agent)
+ CoderFormatInstructions           552       (in /coder)
```

Compact personas often **reduce** the total — `CoderSystemPrompt` (1,637) is larger than many personas + format instructions combined.

## Prompt caching changes the economics completely

The `/agent` and `/coder` system prompt is **stable within a session** — the only dynamic piece is the random `ScratchDir` inside `buildSessionWorkspaceHint` (which is fixed per session).

ChatCLI already marks the system prompt with `CacheControl{Type: "ephemeral"}` for Anthropic (see [Context Commands](/en/features/context-commands)), so caching is automatically on. Price differences:

| Operation | Multiplier | Cost on Sonnet 4.6 |
|:----------|:-----------|-------------------:|
| Regular input | 1.0× | $3.00/M |
| Cache write (1st call) | 1.25× | $3.75/M |
| Cache read (5min TTL) | 0.1× | $0.30/M |

### Practical example — a 30-turn `/coder` session

| Scenario | Calculation | Total cost |
|:---------|:------------|-----------:|
| **No caching** | 30 × 8,343 × $3.00/M | **$0.751** |
| **With caching** (1× write + 29× read) | (1 × 8,343 × $3.75/M) + (29 × 8,343 × $0.30/M) | **$0.104** |

That's **86% reduction** on system prompt spend — and this only counts the fixed prompt, not the growing conversation history.

<Tip>
The cache window is 5 minutes (ephemeral TTL). Sessions with long pauses between messages may lose the cache and pay `cache_write` again. For intensive coding sessions (multiple turns per minute), caching captures essentially everything.
</Tip>

## Where ChatCLI spends the most tokens (and what to cut)

Cost-benefit analysis of each block in `/coder`:

| Block | Tokens | Verdict |
|:------|------:|:--------|
| `OrchestratorSystemPrompt` | **4,073** | 🔴 **Biggest spend.** Lists 12 agents + skills + scripts. Worth gold if you use `<agent_call>`. If you never use multi-agent, turning it off saves ~50% of the system prompt. |
| `getToolContextString` | **2,157** | 🟡 Schemas of 3 built-in plugins. Some overlap with the "OTHER TOOLS" section of `CoderSystemPrompt` (~150 redundant tokens). |
| `CoderSystemPrompt` | **1,637** | 🟢 Required in `/coder` — defines the XML/JSON output contract. |
| `agent.system_prompt.default.base` | **727** | 🟢 Required in `/agent` — defines the base system-admin persona. |
| `buildSessionWorkspaceHint` | **399** | 🟢 Acceptable cost; solves the [tool-result truncation](/en/features/tool-result-management) problem (saves orders of magnitude more on avoided refetches). |
| `ai.response_language` | **77** | 🟢 Trivial. |
| `ChatModeSystemHint` | **214** | 🟢 Trivial — chat mode only. |

## Variables to cut the spend

| Variable | Effect | When it pays off |
|:---------|:-------|:-----------------|
| `CHATCLI_AGENT_PARALLEL_MODE=false` | Removes `OrchestratorSystemPrompt` (~4,073 tokens) | You use `/coder` directly without dispatching `<agent_call>`. Keeps every other tool (read/write/patch/exec/git/test/`delegate_subagent`). |
| `CHATCLI_AGENT_KEEP_TMPDIR=true` | Doesn't change the token budget; keeps the scratch dir after exit for debugging | Investigating agent behaviour |
| Disable unused plugins | Reduces `getToolContextString` proportionally | You don't use `@webfetch`/`@websearch` or custom plugins |
| Compact persona instead of default prompt | Reduces/replaces the main block | Focused personas (e.g. `senior-go`) can be leaner than the generic prompt |

### Concrete example — lean `/coder` session

```bash
CHATCLI_AGENT_PARALLEL_MODE=false chatcli
```

Cuts the system prompt from **8,343 → 4,270 tokens** (-49%). On a 30-turn session with Sonnet 4.6 and caching:

- Before: $0.104
- After: $0.053

## How to reproduce

The measurement is straightforward: ChatCLI exposes the blocks as public Go constants. A small program is enough:

```go
package main

import (
	"fmt"
	"github.com/diillson/chatcli/cli"
	"github.com/diillson/chatcli/cli/agent/workers"
	"github.com/diillson/chatcli/i18n"
)

func tokens(s string) int { return (len(s) + 3) / 4 }

func main() {
	i18n.Init()
	reg := workers.SetupDefaultRegistry()

	blocks := map[string]string{
		"ChatModeSystemHint":      cli.ChatModeSystemHint,
		"CoderSystemPrompt":       cli.CoderSystemPrompt,
		"CoderFormatInstructions": cli.CoderFormatInstructions,
		"AgentFormatInstructions": cli.AgentFormatInstructions,
		"agent.system_prompt.default.base": i18n.T(
			"agent.system_prompt.default.base", "darwin", "zsh", "/proj"),
		"ai.response_language":     i18n.T("ai.response_language"),
		"OrchestratorSystemPrompt": workers.OrchestratorSystemPrompt(reg.CatalogString()),
	}
	for name, s := range blocks {
		fmt.Printf("%-40s %6d chars  ~%5d tokens\n", name, len(s), tokens(s))
	}
}
```

Run with `go run` against the `chatcli` module in your GOPATH or via a replace directive. The **4 chars/token** rule is the industry standard heuristic; for exact precision use `tiktoken-go` for OpenAI models or the Anthropic SDK's own counter.

<Note>
The numbers on this page were collected with Go 1.22 against the `chatcli` `main` branch on 2026-04-14. Changes to `cli/prompts.go` or the agent catalog shift the totals — recompute after version bumps if the spend matters to you.
</Note>
