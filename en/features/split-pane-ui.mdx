The **split-pane UI** (also called **turn UI**) brings a dedicated-line-editor experience to `/coder` mode: the spinner lives in a **pinned status row**, the agent output scrolls inside a **confined region**, and the user always has a **visible input row** at the bottom — inspired by the Claude Code architecture.

Unlike `CHATCLI_CODER_UI`, which controls **how** the agent's boxes/cards are rendered, `CHATCLI_TURNUI` controls **the terminal architecture** (DECSTBM + raw mode + custom mini-readline). The two variables are **orthogonal** — you can combine `CHATCLI_CODER_UI=compact` with `CHATCLI_TURNUI=on` for minimalist cards inside the split-pane.

<Warning>
**Opt-in** feature via `CHATCLI_TURNUI=on`. Default = legacy (single-line spinner). Auto-activation was removed after a real-world cascading-spinner bug on the GoLand integrated terminal, where DECSTBM and cursor save/restore (`ESC 7`/`ESC 8`) were silently ignored.
</Warning>

## Activation

```bash
# Per-session only
CHATCLI_TURNUI=on chatcli

# Or export in your .env / shell rc
export CHATCLI_TURNUI=on
```

Any value other than `on` (empty, `off`, `false`, `1`) keeps the legacy renderer. Activation also requires:

- A real TTY on both stdin **and** stdout (pipes/redirects fall back)
- `TERM` set and not `dumb`
- Terminal of at least 10 rows × 40 cols
- On Windows: console with `ENABLE_VIRTUAL_TERMINAL_PROCESSING` enabled (modern Win10+)

If any gate vetoes, chatcli silently falls back to the legacy renderer — you never end up with a wedged terminal.

## Layout

```
┌────────────────────────────────────────────────────────┐
│  agent output, tool calls, results                    │   ← scroll
│  ...                                                   │     region
│  ↻ Read(main.go)                                       │     1..N-2
│  ✓ Read(main.go) 0.3s                                  │     (DECSTBM)
│  ...                                                   │
├────────────────────────────────────────────────────────┤
│  ⠹ [opus-4-7] [12s] | Processing... (2 queued) ⋯      │   ← status N-1
├────────────────────────────────────────────────────────┤
│  ❯ fix bar.go too█                                     │   ← input  N
└────────────────────────────────────────────────────────┘
```

- **Rows 1..N-2**: confined by DECSTBM (`\x1b[<top>;<bot>r`); agent output scrolls **inside** the region
- **Row N-1**: status with spinner, model, elapsed time, queue depth — redrawn in place via `MoveCursor` + `ClearLine`
- **Row N**: active input in raw mode; the user can type at any time without fighting the spinner

## Keyboard shortcuts

| Key | Action |
| --- | --- |
| `←` / `→` | Move cursor one position in the buffer |
| `Home` / `Ctrl+A` | Jump to start of line |
| `End` / `Ctrl+E` | Jump to end of line |
| `↑` / `↓` | Navigate submission history (FIFO 100 entries) |
| `Backspace` | Delete the rune before the cursor |
| `Delete` | Delete the rune at the cursor |
| `Ctrl+U` | Clear the entire line |
| `Ctrl+W` | Delete the word before the cursor |
| `Ctrl+C` | Clear the buffer (does NOT abort the session; use `/exit`) |
| `Ctrl+D` | On an empty buffer, exits `/coder` |
| `Enter` | Submit the line (lands in the agent's queue) |

History is session-scoped — it survives across turns within the same `/coder` run, but does not persist between processes. Consecutive duplicates are skipped (bash's `HISTCONTROL=ignoreboth`).

## Edge behaviors

**Type-ahead while the LLM streams:** lines typed land in the queue and flash as `📥 queued #N` in the status slot; each turn drains the next.

**Terminal resize:**

- **Linux/macOS**: SIGWINCH handler reflows the layout instantly
- **Windows**: 500 ms polling detects the new dimensions and re-emits DECSTBM

**Security prompts (an `<action>` triggering confirmation):** the split-pane is **suspended** for the duration of the dialog (cooked mode restored, region released), then **resumed** with the status preserved. Without this suspension, arrows and box-drawing leaked as literal `^[[A` into the confirmation box.

**Terminal too small (< 10 rows):** detected in `ShouldActivate` and falls back to legacy automatically. Resizing back up does **not** re-activate mid-turn (an intrusive re-activation would be worse than the transient glitch).

## Compatibility

| Terminal | Status |
| --- | --- |
| iTerm2 (macOS) | ✅ Works |
| Terminal.app (macOS) | ✅ Works |
| Hyper | ✅ Works |
| Alacritty | ✅ Works |
| kitty | ✅ Works |
| GNOME Terminal / Konsole | ✅ Expected to work (xterm-compliant) |
| Windows Terminal | ✅ Expected to work (with ConPTY) |
| **GoLand integrated terminal** | ❌ Does not honor DECSTBM + ESC 7/8 (documented case that motivated the opt-in) |
| **VSCode integrated terminal** | ⚠️ Not tested in production — likely fine |
| **Git Bash (mintty + MSYS2 pty)** | ⚠️ Most likely falls back automatically (`IsTerminal` returns false for MSYS2 pipes); if you activate via `winpty`, it works |
| CI/pipes/dumb-TERM | ⛔ Automatic fallback — escape codes never corrupt logs |

If your terminal claims to "work" in local tests but you see visual glitches (duplicated spinner lines, cursor jumping), open an issue with `TERM`, emulator name, and a short screencast.

## Difference from `CHATCLI_CODER_UI`

| Variable | Controls | Values |
| --- | --- | --- |
| `CHATCLI_CODER_UI` | Card / inline rendering style for the agent | `full`, `minimal`, `compact` |
| `CHATCLI_TURNUI` | Terminal architecture (pinned status + pinned input) | `on` or empty |

They are **orthogonal**. Sensible combinations:

```bash
# Most minimal possible: split-pane + inline cards
CHATCLI_CODER_UI=compact CHATCLI_TURNUI=on chatcli

# Split-pane layout with full cards
CHATCLI_TURNUI=on chatcli

# Current behavior (no split-pane), inline cards
CHATCLI_CODER_UI=compact chatcli
```

## Implementation

Package: [`cli/coder/turnui/`](https://github.com/diillson/chatcli/tree/main/cli/coder/turnui). Layers:

- **`region.go`** — DECSTBM primitives (scroll region, cursor save/restore via ESC 7/8 — more portable than CSI s/u)
- **`keydecode.go`** — raw → `Key` decoder (UTF-8 reassembly, CSI/SS3 swallowing)
- **`linebuf.go`** — rune-aware buffer with cursor for insertion at any position
- **`history.go`** — FIFO 100 stack with `HISTCONTROL=ignoreboth` semantics
- **`readline.go`** — read → decode → apply → paint loop
- **`turnui.go`** — orchestrator (Begin/End/Suspend/Resume/Resize)

See PR `#935` and commit `c0ec063` (opt-in fix) for the design history.
