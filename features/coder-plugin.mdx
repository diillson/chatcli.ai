## Formatos de Argumentos

O `@coder` aceita dois formatos de argumentos: **JSON** e **CLI-style**. Ambos sĂŁo equivalentes.

<Tabs>
  <Tab title="Formato JSON (recomendado)">
    O formato JSON e usado dentro do atributo `args` de um `<tool_call>`. A estrutura e:

    ```json
    {
      "cmd": "<subcomando>",
      "args": {
        "<flag>": "<valor>",
        ...
      }
    }
    ```

    Exemplo completo:
    ```xml
    <tool_call name="@coder" args='{"cmd":"read","args":{"file":"main.go","start":10,"end":50}}'/>
    ```

    <Tip>
    O formato JSON e o recomendado porque evita problemas de escape e e o que os modelos de linguagem geram com mais consistencia.
    </Tip>
  </Tab>

  <Tab title="Formato CLI-style">
    TambĂ©m e possĂ­vel passar argumentos no estilo linha de comando, separados por espaco:

    ```
    read --file main.go --start 10 --end 50
    ```

    Neste caso, o campo `cmd` e a primeira palavra e as flags seguem no formato `--flag valor`.

    Exemplo em tool_call:
    ```xml
    <tool_call name="@coder" args='{"cmd":"read --file main.go --start 10 --end 50"}'/>
    ```

    <Note>
    No formato CLI-style, valores com espacos devem ser colocados entre aspas: `--content "hello world"`.
    </Note>
  </Tab>
</Tabs>

## Sistema de Backup

O `@coder` implementa um sistema de backup automĂˇtico para proteger contra alteraĂ§Ăµes indesejadas.

<Steps>
  <Step title="Escrita ou Patch">
    Quando vocĂŞ executa `write` ou `patch`, o plugin verifica se o arquivo-alvo jĂˇ existe.
  </Step>
  <Step title="Criacao do Backup">
    Se o arquivo existe, uma copia e salva com a extensao `.bak` (ex: `main.go` -> `main.go.bak`).
  </Step>
  <Step title="AplicaĂ§ĂŁo da Alteracao">
    O conteĂşdo novo e escrito (ou o patch e aplicado) no arquivo original.
  </Step>
  <Step title="Rollback DisponĂ­vel">
    A qualquer momento, vocĂŞ pode usar `rollback --file main.go` para restaurar a versĂŁo anterior a partir do `.bak`.
  </Step>
  <Step title="Limpeza">
    Use `clean` para remover todos os arquivos `.bak` quando nĂŁo precisar mais dos backups.
  </Step>
