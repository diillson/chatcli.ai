A **UI split-pane** (também chamada de **turn UI**) traz para o modo `/coder` a experiência de um editor de linha dedicado: o spinner vive numa **linha de status fixa**, o output do agente rola numa **região central confinada**, e o usuário tem uma **linha de input sempre visível** no fundo — inspirada na arquitetura do Claude Code.

Diferente do `CHATCLI_CODER_UI`, que controla **como** as caixas/cards do agente são renderizadas, o `CHATCLI_TURNUI` controla **a arquitetura do terminal** (DECSTBM + modo raw + mini-readline próprio). As duas variáveis são **ortogonais** — você pode combinar `CHATCLI_CODER_UI=compact` com `CHATCLI_TURNUI=on` para ter cards minimalistas no split-pane.

<Warning>
Feature **opt-in** via `CHATCLI_TURNUI=on`. Default = legado (single-line spinner). A ativação automática foi removida após um caso real de cascading-spinner no terminal integrado do GoLand, onde DECSTBM e cursor save/restore (`ESC 7`/`ESC 8`) não eram honrados.
</Warning>

## Ativação

```bash
# Ativa apenas para a sessão atual
CHATCLI_TURNUI=on chatcli

# Ou exporte no seu .env / shell rc
export CHATCLI_TURNUI=on
```

Qualquer valor diferente de `on` (vazio, `off`, `false`, `1`) mantém o renderer legado. A ativação respeita também:

- TTY válido em stdin **e** stdout (pipes/redirects caem em legacy)
- `TERM` setado e diferente de `dumb`
- Terminal de pelo menos 10 linhas × 40 colunas
- Windows: console com `ENABLE_VIRTUAL_TERMINAL_PROCESSING` ativo (Win10+ moderno)

Em qualquer veto, o chatcli cai silenciosamente no renderer legado — você nunca fica com um terminal travado.

## Layout

```
┌────────────────────────────────────────────────────────┐
│  output do agente, tool calls, resultados             │   ← região
│  ...                                                   │     scroll
│  ↻ Read(main.go)                                       │     1..N-2
│  ✓ Read(main.go) 0.3s                                  │     (DECSTBM)
│  ...                                                   │
├────────────────────────────────────────────────────────┤
│  ⠹ [opus-4-7] [12s] | Processando... (2 na fila) ⋯    │   ← status N-1
├────────────────────────────────────────────────────────┤
│  ❯ fix bar.go too█                                     │   ← input  N
└────────────────────────────────────────────────────────┘
```

- **Região 1..N-2**: confinada pelo DECSTBM (`\x1b[<top>;<bot>r`); output do agente rola **dentro** dela
- **Linha N-1**: status com spinner, modelo, tempo decorrido, profundidade da fila — atualiza in-place via `MoveCursor` + `ClearLine`
- **Linha N**: input ativo, em modo raw; o usuário pode digitar a qualquer momento sem brigar com o spinner

## Atalhos de teclado

| Tecla | Acao |
| --- | --- |
| `←` / `→` | Move o cursor uma posicao no buffer |
| `Home` / `Ctrl+A` | Vai para o inicio da linha |
| `End` / `Ctrl+E` | Vai para o fim da linha |
| `↑` / `↓` | Navega o historico de submissões (FIFO 100 entradas) |
| `Backspace` | Apaga o rune anterior ao cursor |
| `Delete` | Apaga o rune sob o cursor |
| `Ctrl+U` | Limpa a linha inteira |
| `Ctrl+W` | Apaga a palavra anterior ao cursor |
| `Ctrl+C` | Limpa o buffer (nao aborta a sessão; use `/sair`) |
| `Ctrl+D` | Em buffer vazio, encerra o `/coder` |
| `Enter` | Submete a linha (vai para a fila do agente) |

O historico é session-scoped — sobrevive entre turns dentro do mesmo `/coder`, mas nao persiste entre processos. Duplicatas consecutivas são puladas (HISTCONTROL=ignoreboth do bash).

## Comportamentos de borda

**Type-ahead enquanto o LLM responde:** linhas digitadas vão para a fila e aparecem como `📥 enfileirado #N` no slot de status temporariamente; cada turn dreina a próxima.

**Resize de terminal:**

- **Linux/macOS**: handler SIGWINCH refaz o layout instantaneamente
- **Windows**: polling de 500ms detecta a nova dimensão e re-emite DECSTBM

**Security prompts (`<action>` que dispara confirmação):** a UI split-pane é **suspensa** durante o dialog (volta a modo cooked, libera a região), depois **retomada** com o status preservado. Sem essa suspensão, setas e box-drawing apareciam como literais `^[[A` na caixa de confirmação.

**Terminal pequeno demais (< 10 linhas):** detectado em `ShouldActivate`, cai no legacy automaticamente. Resize de volta para cima **não** re-ativa no meio do turno (uma re-ativação intrusiva seria pior que o glitch transitório).

## Compatibilidade

| Terminal | Status |
| --- | --- |
| iTerm2 (macOS) | ✅ Funciona |
| Terminal.app (macOS) | ✅ Funciona |
| Hyper | ✅ Funciona |
| Alacritty | ✅ Funciona |
| kitty | ✅ Funciona |
| GNOME Terminal / Konsole | ✅ Esperado funcionar (xterm-compliant) |
| Windows Terminal | ✅ Esperado funcionar (com ConPTY) |
| **GoLand integrated terminal** | ❌ Não honra DECSTBM + ESC 7/8 (caso documentado que motivou o opt-in) |
| **VSCode integrated terminal** | ⚠️ Não testado em produção — provavelmente OK |
| **Git Bash (mintty + MSYS2 pty)** | ⚠️ Provavelmente cai no fallback automatico (`IsTerminal` retorna false em pipe MSYS2); se ativar via `winpty`, funciona |
| CI/pipes/dumb-TERM | ⛔ Fallback automatico — escape codes nao corrompem logs |

Se o seu terminal aparecer como "funciona" em testes locais mas com glitches visuais (linhas duplicadas de spinner, cursor pulando), abra um issue com `TERM`, nome do emulador, e um screencast curto.

## Diferenca para `CHATCLI_CODER_UI`

| Variavel | Controla | Valores |
| --- | --- | --- |
| `CHATCLI_CODER_UI` | Estilo dos cards/inline do agente | `full`, `minimal`, `compact` |
| `CHATCLI_TURNUI` | Arquitetura do terminal (status fixo + input fixo) | `on` ou vazio |

São **ortogonais**. Combinações sensatas:

```bash
# Mais minimalista possível: split-pane + cards inline
CHATCLI_CODER_UI=compact CHATCLI_TURNUI=on chatcli

# Layout split-pane mas cards completos
CHATCLI_TURNUI=on chatcli

# Comportamento atual (sem split-pane), cards inline
CHATCLI_CODER_UI=compact chatcli
```

## Implementação

Pacote: [`cli/coder/turnui/`](https://github.com/diillson/chatcli/tree/main/cli/coder/turnui). Camadas:

- **`region.go`** — primitivas DECSTBM (scroll region, cursor save/restore via ESC 7/8 — mais portável que CSI s/u)
- **`keydecode.go`** — decoder raw → `Key` (UTF-8 reassembly, CSI/SS3 swallowing)
- **`linebuf.go`** — buffer rune-aware com cursor para inserção em qualquer posição
- **`history.go`** — stack FIFO 100 com semantica HISTCONTROL=ignoreboth
- **`readline.go`** — loop read → decode → apply → paint
- **`turnui.go`** — orquestrador (Begin/End/Suspend/Resume/Resize)

Veja o PR `#935` e o commit `c0ec063` (fix opt-in) para o histórico de design.
