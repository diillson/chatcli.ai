### 3. CorrelationEngine (`correlation.go`)

Motor de correlaĂ§ĂŁo que agrupa anomalias em incidentes.

**Algoritmo de CorrelaĂ§ĂŁo:**

```text
Para cada nova anomalia:
  1. Gera incident_id = hash(resource_kind + resource_name + namespace + signal_type)
  2. Busca Issue existente com mesmo incident_id
  3. Se existe -> adiciona anomalia ao Issue, recalcula risk score
  4. Se nĂŁo existe -> cria novo Issue
```

**Risk Scoring:**

| Sinal | Peso | Justificativa |
|-------|------|---------------|
| `oom_kill` | 30 | Indica problema de memĂłria severo |
| `error_rate` | 25 | Impacto direto em usuĂˇrios |
| `deploy_failing` | 25 | Indisponibilidade do serviĂ§o |
| `latency_spike` | 20 | DegradaĂ§ĂŁo de performance |
| `pod_restart` | 20 | Instabilidade do pod |
| `pod_not_ready` | 20 | Capacidade reduzida |

**ClassificaĂ§ĂŁo de Severidade:**

```text
risk_score >= 80 -> Critical
risk_score >= 60 -> High
risk_score >= 40 -> Medium
risk_score <  40 -> Low
```

**Exemplo**: Um deployment com `oom_kill` (30) + `pod_restart` (20) = risk 50 â†’ **Medium**. Se adicionar `error_rate` (25) = risk 75 â†’ **High**.
