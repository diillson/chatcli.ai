A **API REST do AIOps Platform** permite integrar qualquer sistema externo com a plataforma de operações autônomas do ChatCLI. Todos os endpoints seguem convenções RESTful e retornam JSON.

A API é servida pelo Web Dashboard na porta `8090` (configurável via `CHATCLI_AIOPS_PORT`).

---

## Autenticação

<AccordionGroup>
  <Accordion title="X-API-Key Header">
    Todas as requisições autenticadas devem incluir o header `X-API-Key`:

    ```bash
    curl -H "X-API-Key: seu-token-aqui" http://localhost:8090/api/v1/incidents
    ```

    As chaves são configuradas via ConfigMap `chatcli-api-keys` no namespace do operator:

    ```yaml
    apiVersion: v1
    kind: ConfigMap
    metadata:
      name: chatcli-api-keys
    data:
      keys.json: |
        {
          "keys": [
            {"key": "ops-key-abc123", "role": "operator", "description": "CI/CD pipeline"},
            {"key": "view-key-xyz789", "role": "viewer", "description": "Dashboard read-only"},
            {"key": "admin-key-root01", "role": "admin", "description": "Platform admin"}
          ]
        }
    ```
  </Accordion>

  <Accordion title="Roles e Permissões">
    | Role | Leitura | Ack/Snooze | Aprovar/Rejeitar | CRUD Runbooks | Audit Export |
    |------|---------|------------|-----------------|---------------|-------------|
    | `viewer` | Sim | Nao | Nao | Nao | Nao |
    | `operator` | Sim | Sim | Sim | Nao | Nao |
    | `admin` | Sim | Sim | Sim | Sim | Sim |
  </Accordion>

  <Accordion title="Modo Desenvolvimento">
    Quando nenhuma chave de API está configurada (ConfigMap ausente), a API opera em **modo dev** — todas as requisições são permitidas sem autenticação.

    <Warning>
      Nunca use modo dev em produção. Configure ao menos uma chave de API antes de expor o serviço.
    </Warning>
  </Accordion>

  <Accordion title="Rate Limiting">
    A API aplica rate limiting de **100 requisições por minuto** por IP.

    Quando o limite é excedido, a API retorna:

    ```json
    {
      "error": "rate limit exceeded",
      "retry_after_seconds": 42
    }
    ```

    | Header | Descrição |
    |--------|-----------|
    | `X-RateLimit-Limit` | Limite por janela (100) |
    | `X-RateLimit-Remaining` | Requisições restantes |
    | `X-RateLimit-Reset` | Timestamp Unix do reset da janela |
    | `Retry-After` | Segundos até o reset (apenas quando excedido) |
  </Accordion>
</AccordionGroup>

## Formato de Resposta

Todas as respostas seguem o envelope padrão:

**Sucesso (item único):**
```json
{
  "data": { ... }
}
```

**Sucesso (lista com paginação):**
```json
{
  "data": [ ... ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "total": 142,
    "totalPages": 8
  }
}
```

**Erro:**
```json
{
  "error": "mensagem descritiva do erro",
  "code": "NOT_FOUND"
}
```

## Health

Endpoints de health check não requerem autenticação.

### GET /healthz

Verifica se o servidor está vivo.

<ParamField header="Authorization" type="none">
  Nenhuma autenticação necessária.
</ParamField>

**Resposta `200 OK`:**

```json
{
  "status": "ok",
  "timestamp": "2026-03-19T14:30:00Z"
}
```

```bash
curl http://localhost:8090/healthz
```

### GET /readyz

Verifica se o servidor está pronto para receber tráfego (conectado ao cluster, reconcilers ativos).

**Resposta `200 OK`:**

```json
{
  "status": "ready",
  "checks": {
    "kubernetes": "connected",
    "reconcilers": "running",
    "grpc_server": "listening"
  },
  "timestamp": "2026-03-19T14:30:00Z"
}
```

**Resposta `503 Service Unavailable`:**

```json
{
  "status": "not_ready",
  "checks": {
    "kubernetes": "connected",
    "reconcilers": "starting",
    "grpc_server": "not_listening"
  },
  "timestamp": "2026-03-19T14:30:00Z"
}
```

```bash
curl http://localhost:8090/readyz
```

## Incidents

Gerenciamento de incidentes detectados pelo pipeline AIOps.

### GET /api/v1/incidents

Lista todos os incidentes com filtros e paginação.

**Role mínimo:** `viewer`

<ParamField query="severity" type="string">
  Filtrar por severidade. Valores: `critical`, `high`, `medium`, `low`.
</ParamField>

<ParamField query="state" type="string">
  Filtrar por estado. Valores: `detected`, `analyzing`, `remediating`, `resolved`, `escalated`.
</ParamField>

<ParamField query="namespace" type="string">
  Filtrar por namespace Kubernetes.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Número da página.
</ParamField>

<ParamField query="pageSize" type="integer" default="20">
  Itens por página. Máximo: 100.
</ParamField>

<ParamField query="from" type="string">
  Data/hora de início no formato ISO 8601. Ex: `2026-03-01T00:00:00Z`.
</ParamField>

<ParamField query="to" type="string">
  Data/hora de fim no formato ISO 8601.
</ParamField>

**Resposta `200 OK`:**

}

```json

    "name": "issue-crashloop-api-server-production",
    "snoozedUntil": "2026-03-19T15:35:00Z",

  }
}
```

```bash
curl -X POST -H "X-API-Key: ops-key-abc123" \
  -H "Content-Type: application/json" \
  -d '{"duration": "1h"}' \
  "http://localhost:8090/api/v1/incidents/issue-crashloop-api-server-production/snooze"
```

O header `Content-Type` será `text/csv` e o body conterá o CSV com colunas:
`id,timestamp,type,severity,actor,resource,namespace,description`

```bash
# Exportar como JSON
curl -H "X-API-Key: admin-key-root01" \
  "http://localhost:8090/api/v1/audit/export?format=json&from=2026-03-01T00:00:00Z" \
  -o audit-export.json

# Exportar como CSV
curl -H "X-API-Key: admin-key-root01" \
  "http://localhost:8090/api/v1/audit/export?format=csv&from=2026-03-01T00:00:00Z" \
  -o audit-export.csv
```

## Códigos de Erro

| Código HTTP | Código | Descrição |
|-------------|--------|-----------|
| `400` | `BAD_REQUEST` | Parâmetro inválido ou corpo malformado |
| `401` | `UNAUTHORIZED` | API key ausente ou inválida |
| `403` | `FORBIDDEN` | Role insuficiente para a operação |
| `404` | `NOT_FOUND` | Recurso não encontrado |
| `409` | `CONFLICT` | Recurso já existe (ex: runbook com mesmo nome) |
| `429` | `RATE_LIMITED` | Rate limit excedido |
| `500` | `INTERNAL_ERROR` | Erro interno do servidor |
| `503` | `SERVICE_UNAVAILABLE` | Servidor não está pronto (readyz falhou) |
